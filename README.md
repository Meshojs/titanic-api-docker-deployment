# Titanic Survival Predictor API :> 🐳

https://masharyy-titanic-api.hf.space/

A machine learning API that predicts Titanic passenger survival, built with **FastAPI** and containerized with **Docker** — enabling a clean **CI/CD** deployment pipeline.

## Overview

This project takes a trained PyTorch model and serves it through a production-style REST API. FastAPI handles request validation and docs, while Docker packages the whole app (model, preprocessing, and API) into a portable image ready for CI/CD.

## Tech Stack

- **PyTorch** — model training & inference
- **FastAPI** — REST API with Pydantic request/response validation
- **Docker** — containerization for consistent, reproducible deployment
- **CI/CD** — automated build & deploy pipeline enabled by the Docker image

## Features

- `/predict` — returns survival prediction from passenger data
- `/health` — health check endpoint
- `/docs` — interactive Swagger UI (auto-generated by FastAPI)

## Why FastAPI + Docker

FastAPI made it straightforward to define typed request/response schemas and get automatic API documentation for free. Wrapping the app in Docker meant the model, preprocessing pipeline, and API server all ship as one image ; which is what made a CI/CD pipeline possible, since the same image built in CI is exactly what gets deployed.

## Running Locally

```bash
docker build -t titanic-api .
docker run -p 8000:8000 titanic-api
```

Then visit `http://localhost:8000/docs` for the interactive API docs.
